RNN (Recurrent Neural Network): 순차 데이터 처리를 위한 신경망

1. RNN이란 무엇인가?

Recurrent Neural Network(RNN)는 순차적 데이터(Sequential Data)를 처리하도록 설계된 신경망입니다. 일반적인 신경망(예: Fully Connected Layer)은 고정된 입력과 출력을 다루는 데 비해, RNN은 시간에 따라 변화하는 데이터(예: 시계열 데이터, 자연어 텍스트)를 효과적으로 학습할 수 있는 구조를 가지고 있습니다.

2. 왜 RNN이 필요한가?

문맥의 중요성

문장을 예로 들어 보겠습니다.

다음과 같은 문장을 이해하려면 무엇이 필요할까요?

"나는 오늘도 아침에 커피를 마셨다."

이 문장은 단어 하나하나가 독립적으로 의미를 가지는 것이 아니라, 문맥(Context)에 따라 단어의 의미가 결정됩니다.

예를 들어, "커피"의 의미는 이전 단어 "마셨다"와 연결되며, 문맥을 이해해야 전체 의미를 파악할 수 있습니다.

기존 신경망(Fully Connected Layer)의 한계

기존의 Fully Connected Layer는 모든 입력 데이터를 독립적으로 처리하기 때문에 시간 순서나 문맥 정보를 반영하지 못합니다.

예:

"나는", "오늘", "커피"라는 단어를 순서 없이 처리하면, 이 단어들의 연관성을 알 수 없습니다.

결과적으로, 문장의 의미를 제대로 학습할 수 없습니다.

3. RNN의 기본 구조와 작동 방식

RNN의 구조

RNN은 이전 단계의 정보를 현재 단계로 전달하는 순환 구조를 가지고 있습니다. 이를 통해 입력 데이터의 시간적 순서와 문맥을 학습할 수 있습니다.

RNN의 핵심 요소는 다음과 같습니다. RNN은 이야기를 읽으면서 앞에서 읽었던 내용을 기억하고, 이를 바탕으로 다음 내용을 이해하는 사람처럼 작동합니다.

입력 데이터 (xtx_txt​﻿﻿) : 시간 t에서의 입력 데이터

"나는 아침에 커피를 마셨다"라는 문장을 읽는다고 가정합니다.

RNN은 한 번에 한 단어씩 읽으면서 그 의미를 학습합니다.

은닉 상태 (hth_tht​﻿): 현재까지 학습된 정보를 담고 있는 내부 상태.

이전에 읽었던 내용을 머릿속에서 정리해두는 "기억" 역할을 합니다.

예를 들어, "나는"을 읽으면, 이를 기억합니다. 이어 "아침에"를 읽을 때 "나는"이라는 정보를 바탕으로 맥락을 이해합니다.

출력 데이터 (yty_tyt​﻿﻿): 시간 t에서의 출력 결과.

각 단어를 처리한 뒤, 그 단어에 대한 정보를 바탕으로 예측을 합니다.

예를 들어, "커피를"까지 읽었을 때, "마셨다"라는 단어가 올 가능성을 높게 평가합니다.

2. RNN의 작동 원리: 단계별 흐름

RNN은 마치 누군가가 그림의 일부를 그리고, 다음 사람이 이어서 그림을 완성하는 작업처럼 작동합니다.

이 작업에서 이전에 그린 그림(은닉 상태)은 다음 작업(다음 단어 처리)의 참고 자료가 됩니다.

예시) 나는 아침에 커피를 마셨다.

처음 단어를 처리 (입력 xix_ixi​﻿﻿)

첫 번째 단어 "나는"을 읽고, 이 단어를 해석한 후 기억합니다.

이 기억은 첫 번째 은닉 상태 h1h_1h1​﻿﻿에 저장됩니다.

다음 단어를 처리 (입력 x2x_2x2)

두 번째 단어 "아침에"를 읽을 때, 첫 번째 기억 h1h1﻿을 참고합니다.

"나는"과 "아침에"를 조합해 더 나은 이해를 얻고, 새로운 은닉 상태 h2h2﻿를 생성합니다.

마지막 단어까지 반복

마지막 단어 "마셨다"까지 반복하며 모든 단어를 연결해 문맥을 이해합니다.

이 최종 은닉 상태는 문장의 전체 맥락을 포함하게 됩니다.

3. 가중치 공유

RNN의 중요한 특징은 가중치 공유(Weight Sharing)입니다.

모든 시점에서 동일한 가중치 Wh,WxW_h,W_xWh​,Wx​﻿﻿를 사용합니다.

이를 통해 입력 데이터의 길이에 관계없이 일관된 학습이 가능하며, 메모리 효율성도 높습니다.

4. RNN의 한계

장기 의존성 문제 (Vanishing Gradient Problem)

RNN은 시간이 긴 데이터를 처리할 때 장기 의존성(Long-Term Dependency) 문제를 겪습니다.

문제 설명:

초기 입력 데이터가 후반부에 도달할수록 영향력이 희미해지는 문제가 발생합니다.

예를 들어:

문장 "나는 아침에 커피를 마셨다"에서 "나는"이라는 단어의 정보가 "마셨다"를 학습하는 단계에서 유실될 수 있습니다.

원인:

RNN의 학습 과정에서 역전파(Backpropagation)를 수행할 때, 기울기(Gradient)가 시간이 지남에 따라 매우 작아져 Vanishing Gradient 문제가 발생합니다.

결과:

긴 문맥 정보를 학습하는 데 어려움을 겪게 됩니다.

계산 복잡성 증가

시간 순서 데이터는 각 단계마다 연속적으로 계산이 필요하기 때문에, 데이터 길이가 길어질수록 계산량이 증가합니다.

5. RNN의 요약: 이야기와 기억의 조화

구조적 비유:

RNN은 사람이 문장을 읽고 기억하며 다음 단어를 예측하는 과정과 비슷합니다.

각 단어를 처리한 뒤, 이를 은닉 상태에 저장해 다음 단어 처리에 활용합니다.

가중치 공유:

모든 단어를 동일한 규칙으로 처리하여 일관성을 유지합니다.

한계:

긴 문장에서 초반의 내용을 잊어버릴 수 있습니다. 이는 사람도 긴 이야기를 들을 때 중간에 앞부분을 잊는 것과 비슷합니다.

Quiz

문제 1: RNN이 기존 신경망(Fully Connected Layer)과 비교해 가지는 주요 차이점은 무엇인가요?

  • 답변:

    RNN은 입력 데이터의 시간 순서와 문맥을 학습할 수 있도록 설계된 순환 구조를 가지고 있습니다.

    Fully Connected Layer는 모든 입력 데이터를 독립적으로 처리하지만, RNN은 이전 단계의 은닉 상태(ht−1h_{t-1}ht−1​﻿)를 다음 단계로 전달하여, 순차적 데이터의 연관성을 반영할 수 있습니다.

문제 2: RNN이 장기 의존성 문제(Vanishing Gradient Problem)를 겪는 이유는 무엇인가요?

  • 답변:

    RNN은 긴 데이터를 처리할 때, 역전파 과정에서 기울기(Gradient)가 매우 작아지거나 사라지는 Vanishing Gradient 문제가 발생합니다.

    이로 인해 초반 입력 데이터의 정보가 후반부로 전달되지 못해, 긴 문맥 정보를 학습하기 어렵게 됩니다.

import torch
import torch.nn as nn

# RNN 모델 정의
class SimpleRNN(nn.Module):
    def __init__(self, input_size, hidden_size, output_size):
        super(SimpleRNN, self).__init__()
        self.rnn = nn.RNN(input_size, hidden_size, batch_first=True)
        self.fc = nn.Linear(hidden_size, output_size)

    def forward(self, x):
        h0 = torch.zeros(1, x.size(0), hidden_size)  # 초기 은닉 상태
        out, _ = self.rnn(x, h0)  # RNN 계산
        out = self.fc(out[:, -1, :])  # 최종 출력 계산
        return out

# 모델 초기화
input_size = 10
hidden_size = 20
output_size = 1
model = SimpleRNN(input_size, hidden_size, output_size)

# 샘플 입력 데이터
x = torch.randn(32, 5, input_size)  # 배치 크기=32, 시퀀스 길이=5
output = model(x)
print(output.size())  # 출력 크기: (32, 1)
