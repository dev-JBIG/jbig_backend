전처리 : 데이터 인코딩

데이터 인코딩

  • 개념

    데이터 인코딩은 데이터를 특정 형식으로 변환하는 과정을 의미한다.컴퓨터는 숫자만 이해할 수 있기 때문에 자연어로 이뤄진 데이터는 숫자로 변환하는 과정이 필요하다. 이를 Encoding이라고 한다

  • 종류
    • Label Encoding

      Label Encoding은 범주형(Categorical) 변수를 숫자로 변환하는 방법 중 하나로, 각 Category를 고유한 정수로 대응시킨다.
      예컨대 '봄', '여름', '가을', '겨울' 로 이뤄진 계절 Category를 Label Encoding하면 '봄'은 0, '여름'은 1, '가을'은 2, '겨울'은 3으로 변환된다.
      이 방법은 범주형 변수를 편리하게 숫자로 변환할 수 있으나, 특정 머신러닝 모델에서는 변환된 정수를 Category 간의 우열로 판단하거나 가중치 등으로 확대 해석 할 여지가 있다.

      • 코드
      from sklearn.preprocessing import LabelEncoder
items=['TV','냉장고','전자레인지','컴퓨터','선풍기','선풍기','믹서','믹서']
# LabelEncoder를 객체로 생성한 후 , fit( ) 과 transform( ) 으로 label 인코딩 수행.
encoder = LabelEncoder()
encoder.fit(items)
labels = encoder.transform(items)
print('인코딩 변환값:',labels)

    • One-Hot Encoding

      One-Hot Encoding도 범주형(Categorical) 변수를 숫자로 변환하는 방법 중 하나로, Label Encoding과 달리 각 Category를 같은 크기의 벡터로 변환한다.

      예를 들어, '봄', '여름', '가을', '겨울'로 이뤄진 계절 Category를 OneHot Encoding하면 '봄'은 [1, 0, 0, 0], '여름'은 [0, 1, 0, 0], '가을'은 [0, 0, 1, 0], '겨울'은 [0, 0, 0, 1]으로 변환된다.

      이 방식은 각 Category가 같은 크기의 독립적인 벡터로 변환되기 때문에, 순서나 우열, 중요도 등으로 확대 해석될 여지가 없다.

      One-Hot Encoding은 Category의 개수 만큼의 크기를 갖는 벡터를 생성하게 되므로 많은 메모리를 차지한다는 문제가 있다. 예를 들어 100가지 종류의 메뚜기로 이뤄진 categorical column을 One-Hot Encoding 한다면, 크기가 100인 벡터가 row의 수 만큼 메모리에 존재하게 된다. 이는 매우 비효율적이다.

      • 코드

        아래 두개의 코드 결과는 같습니다. fit, transform을 따로 적용해야하는 상황을 제외하고는 보통 get_dummies를 많이 사용합니다.

        from sklearn.preprocessing
import OneHotEncoder
import numpy as np

items=['TV','냉장고','전자레인지','컴퓨터','선풍기','선풍기','믹서','믹서']

# 2차원 ndarray로 변환합니다.
items = np.array(items).reshape(-1, 1)

# 원-핫 인코딩을 적용합니다.
oh_encoder = OneHotEncoder()
oh_encoder.fit(items)
oh_labels = oh_encoder.transform(items)

# OneHotEncoder로 변환한 결과는 희소행렬이므로 toarray()를 이용해 밀집 행렬로 변환.
print('원-핫 인코딩 데이터')
print(oh_labels.toarray())
print('원-핫 인코딩 데이터 차원')print(oh_labels.shape)
        import pandas as pd

df = pd.DataFrame({'item':['TV','냉장고','전자레인지','컴퓨터','선풍기','선풍기','믹서','믹서'] })
pd.get_dummies(df)

  • (Scikit-learn) fit / transform

    전처리에 주로 사용되는 라이브러리 Scikit-learn으로 데이터를 변환하는 과정은 fit과 transform으로 이뤄진다.

    우선 Encoder의 선언을 통해 어떤 Encoding 방식을 사용할 것인지 정한다.

    1. fit

      Encoder에 원본 데이터와 변환 후 데이터의 대응 관계를 학습시킨다.

      여기서 '학습'이란, 데이터의 구조를 이해하고, 이를 바탕으로 데이터를 변환하기 위한 초기 설정을 마련하는 과정을 말한다.

      LabelEncoder하는 과정을 예로 들어보자.

      from sklearn.preprocessing import LabelEncoder

items=['TV','냉장고','전자레인지','컴퓨터','선풍기','선풍기','믹서','믹서']

# LabelEncoder를 객체로 생성한 후 , fit( ) 과 transform( ) 으로 label 인코딩 수행.
encoder = LabelEncoder()
encoder.fit(items)

      위 코드를 실행하면 LabelEncoder 방식을 사용하는 encoder에, ‘TV'는 0, '냉장고'는 1, '전자레인지'는 2, '컴퓨터'은 3 등으로 변환한다는 규칙이 담긴다.

    1. transform

      fit 메소드를 통해 학습된 규칙을 기반으로 실제 데이터를 변환한다.

      labels = encoder.
transform(items)print('인코딩 변환값:',labels)

      위 코드를 실행하면 fit 메소드를 통해 학습된 규칙에 따라 data가 변환된다.

      실행 결과는 다음과 같다.

    1. fit_transform

      fit과 transform을 한번에 진행해줍니다.

      필요한 매개변수를 학습하고 데이터에 즉시 변환을 적용합니다.

      # Initialize the LabelEncoder
label_encoder = LabelEncoder()

df['Category_encoded'] = label_encoder.fit_transform(df['Category'])

      Scikit-learn의 fit / transform은 Encoding 뿐만 아니라 후술할 Scaling에도 같은 방식으로 사용되니 잘 알아두자.
