# 스토리지 비공개(프라이버시) 모델

이 문서는 업로드 파일(첨부/본문 이미지)의 공개 범위를 어떻게 통제하는지, 아직 무엇이
공개로 남아 있는지, 그리고 파일을 "진짜 비공개"로 만들려면 어떤 인프라 작업이 필요한지를
정리한다.

## 현재 모델

- **스토리지**: Cloudflare R2(오브젝트 스토리지)를 사용한다. R2는 커스텀 도메인/CDN에
  연결되면 **도메인 단위로 공개**된다. 즉 객체별 ACL로 개별 파일을 막는 방식이 아니라,
  URL(키)을 아는 사람은 누구나 파일을 받을 수 있다.
- **추측 불가 키**: 업로드 경로는 `uploads/<년>/<월>/<일>/<user_id>/<uuid>.<ext>` 형식으로,
  UUID 때문에 키를 무작위로 맞혀서 접근하는 것은 사실상 불가능하다. 다만 이는 URL이
  유출되지 않는 한에서의 방어(보안상 "은닉"에 가깝다)일 뿐, 접근 통제 자체는 아니다.
- **앱은 게이트 대상 콘텐츠에 공개 URL을 절대 내보내지 않는다**: 게시판 공개범위가
  `member`/`staff`이거나 글 유형이 비공개(스태프전용/사유서)면 아래 두 경로로만 파일이
  나간다. 판정은 `boards/serializers.py`의 `is_post_media_gated(post)` 한 곳으로 통일해,
  첨부와 본문 이미지가 서로 다르게 판정되어 한쪽만 새는 일을 막는다.
  - **첨부파일**: 권한을 확인하는 백엔드 다운로드 엔드포인트로만 내려간다
    (`PostAttachmentDownloadView`, `/api/posts/<id>/attachments/<index>/download/`).
    로그인/게시판 권한을 직접 검사한 뒤 백엔드가 바이트를 스트리밍하며, 원본 키/URL은
    클라이언트에 노출하지 않는다.
  - **본문 인라인 이미지**: 브라우저의 `<img src>`는 Authorization 헤더를 실을 수 없으므로,
    권한 판정 결과를 담은 **6시간 만료 서명 토큰**을 URL에 넣어 내려보낸다
    (`MediaStreamView`, `/api/media/stream/?token=...`). 토큰은
    `django.core.signing`(salt=`jbig.media.stream`)으로 서명·타임스탬프되며, 위조·만료된
    토큰은 403으로 거부된다. 이미지는 페이지 내에서 렌더되어야 하므로 `Content-Disposition:
    inline`으로 응답한다.

## 아직 공개로 남아 있는 것

- **공개(`all`) 게시판의 일반 글 미디어**: 첨부/본문 이미지 모두 고정 공개 URL
  (`public_media_url`)로 내려간다. CDN 캐시가 동작하도록 의도된 것이며, 애초에 공개
  콘텐츠이므로 문제되지 않는다.
- **과거에 유출된 URL**: 위 게이트는 "앱이 공개 URL을 새로 내보내지 않는다"는 보장일 뿐,
  R2 버킷이 도메인 단위로 공개인 한, **이미 외부로 알려진 키(URL)는 여전히 직접 접근이
  가능하다**. 비공개로 전환하기 전에 유출된 링크는 앱 단에서 되돌릴 수 없다.

## 파일을 "진짜 비공개"로 만들려면 (인프라 작업)

앱 레벨 게이트만으로는 "키를 아는 사람"을 막지 못한다. 근본적으로 막으려면 스토리지/CDN
설정을 함께 손봐야 하며, 다음 두 방향 중 하나를 택한다.

1. **공개 게시판 미디어만 CDN 도메인으로 서빙**: 버킷을 기본 비공개로 두고, 공개(`all`)
   게시판에서 쓰는 키만 별도 접두사/버킷으로 분리해 CDN 공개 도메인에 태운다. 나머지
   비공개 파일은 CDN 도메인에서 접근 불가가 된다.
2. **버킷을 완전 비공개로 전환하고 모든 접근을 백엔드 스트리밍으로만**: 공개 파일까지
   포함해 모든 다운로드를 백엔드(다운로드 엔드포인트 / 서명 토큰 스트림)를 거치게 한다.
   접근 통제는 가장 강력하지만, **모든 미디어 트래픽이 백엔드를 통과하므로 CDN 캐시 이점이
   사라지고 백엔드 egress(전송) 비용과 부하가 늘어난다**는 트레이드오프가 있다.

두 방식 모두 앱 코드는 이미 준비되어 있고(공개 URL 미노출 + 게이트 스트리밍), 남은 것은
버킷 공개 정책과 CDN 라우팅을 조정하는 인프라 작업이다.
